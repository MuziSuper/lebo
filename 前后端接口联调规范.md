# 大模型技能文档：前后端接口联调规范

## 一、技能概述

本技能针对项目前后端接口联调场景，统一规范后端接口返回数据格式，明确异常响应、正常响应、分页响应的字段规则、数据类型和对应实体类，助力前后端高效对接，降低联调沟通成本，保证接口数据返回的一致性和规范性，避免数据格式错乱、字段解析失败等问题。

## 二、接口返回数据整体规范

项目后端接口返回数据分为三类场景，分别为异常错误返回、正常成功返回、分页接口返回，三类格式相互独立、互不冲突，后端通过统一处理器处理响应，前端可根据响应状态码和返回字段快速区分响应类型，完成数据解析和业务处理。

## 三、详细返回格式说明

### （一）异常错误返回格式

当后端接口运行抛出异常时，由专属的ExceptionApi异常处理器统一拦截处理，自动封装异常响应数据返回，此类响应的HTTP状态码非200，固定返回标准化的异常响应体。

#### 1. JSON返回示例

```json
{
    "status": 500,
    "message": "错误描述信息",
    "time": "2024-01-01T12:00:00"
}
```

#### 2. 对应实体类

ExceptionResponse

#### 3. 字段明细说明

|字段|数据类型|字段说明|
|---|---|---|
|status|int|业务错误状态码，标识具体异常类型|
|message|String|错误描述信息，直白说明异常原因，便于前端提示和问题排查|
|time|LocalDateTime|接口响应时间，格式为标准时间戳|
### （二）正常成功返回格式

接口请求正常、业务逻辑执行无异常时，返回该格式数据，HTTP状态码为200，返回体封装业务数据和基础响应信息，适配各类单值、单对象等非分页接口。

#### 1. JSON返回示例

```json
{
    "data": <业务数据>,
    "error": null,
    "time": "2024-01-01T12:00:00"
}
```

#### 2. 对应实体类

Result<D>（泛型类，D为具体业务数据类型）

#### 3. 字段明细说明

|字段|数据类型|字段说明|
|---|---|---|
|data|D（泛型）|核心业务数据，根据接口业务场景不同，返回对应类型的数据|
|error|String|错误信息字段，接口正常响应时固定为null，无报错信息|
|time|LocalDateTime|接口响应时间，格式为标准时间戳|
### （三）分页接口返回格式

针对列表查询、数据分页展示类接口，沿用正常成功返回的外层格式，data字段替换为MyBatis-Plus的分页对象IPage<T>，封装分页参数和列表数据，满足前端分页渲染需求。

#### 1. JSON返回示例

```json
{
    "data": {
        "records": [<数据列表>],
        "total": 100,
        "size": 10,
        "current": 1,
        "pages": 10
    },
    "error": null,
    "time": "2024-01-01T12:00:00"
}
```

#### 2. 对应实体类

IPage<T>（MyBatis-Plus官方分页接口，T为列表数据实体类）

#### 3. 字段明细说明

|字段|数据类型|字段说明|
|---|---|---|
|records|List<T>|当前页码对应的业务数据列表，为分页展示的核心数据|
|total|long|数据总记录数，用于前端计算总页数、展示数据总量|
|size|long|每页展示的数据条数，即分页大小|
|current|long|当前查询的页码数，从1开始计数|
|pages|long|数据总页数，由总记录数和分页大小计算得出|

## 五、联调注意事项

1. 前端对接时，先通过HTTP状态码区分正常响应和异常响应，状态码200则解析正常返回格式，非200则解析异常错误格式。

2. 正常接口中error字段固定为null，前端无需做错误提示，只需解析data字段数据。

3. 分页接口严格遵循IPage分页字段规范，前端根据total、current、pages等字段实现分页控件渲染。

4. 前后端需统一实体类字段命名和数据类型，避免字段解析失败、数据类型不匹配等问题。
