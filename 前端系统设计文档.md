# 乐播微信小程序 - 前端系统设计文档

## 1. 项目概述

### 1.1 项目名称
乐播微信小程序 (lebo_wxapp)

### 1.2 项目简介
乐播微信小程序是一款积分商城类应用，用户可以通过签到、购物等方式获取积分，并使用积分兑换商品。应用采用 Taro 跨平台框架开发，支持微信小程序等多端运行。

### 1.3 技术栈

| 技术类别 | 技术名称 | 版本 | 说明 |
|---------|---------|------|------|
| 框架 | Taro | 4.1.9 | 跨平台小程序框架 |
| 前端框架 | Vue | ^3.3.0 | 渐进式 JavaScript 框架 |
| 组件库 | NutUI | ^4.2.8 | Taro 专用组件库 |
| 样式预处理器 | Sass | - | CSS 预处理器 |
| 抽奖组件 | @lucky-canvas/taro | ^0.0.15 | 幸运大转盘组件 |

## 3. 功能模块设计

### 3.1 用户模块

#### 3.1.1 登录注册

**流程说明**
1. 用户点击登录，获取微信授权码（jscode）
2. 调用 `/user/login` 接口验证用户是否已注册
3. 若已注册，返回 token 并跳转首页
4. 若未注册，提示用户完善信息并跳转到注册页面

**登录接口**

| 项目 | 说明 |
|------|------|
| URL | /user/login |
| Method | POST |
| 参数 | jscode (微信授权码) |
| 认证 | 不需要 |
| 返回 | Authorization token |

**注册接口**

| 项目 | 说明 |
|------|------|
| URL | /user/register |
| Method | POST |
| 参数 | jscode, nickName, avatarUrl, gender |
| 认证 | 不需要 |
| 返回 | Authorization token |

#### 3.1.2 用户信息

**获取用户信息接口**

| 项目 | 说明 |
|------|------|
| URL | /user/info |
| Method | GET |
| 认证 | 需要 (Authorization token) |

**响应字段**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| nickName | String | 用户昵称 |
| avatar | String | 头像URL |
| accumulatedPoint | Long | 累计积分 |
| gender | Integer | 性别：0-保密，1-男，2-女 |
| city | String | 城市 |
| email | String | 邮箱 |
| age | Integer | 年龄 |
| phone | String | 手机号 |
| birthday | String | 生日 |
| statusCode | Integer | 账户状态 |

**更新用户信息接口**

| 项目 | 说明 |
|------|------|
| URL | /user/update |
| Method | POST |
| 认证 | 需要 |

#### 3.1.3 签到功能

| 项目 | 说明 |
|------|------|
| URL | /signin |
| Method | POST |
| 认证 | 需要 |

## 6. 认证机制

### 6.1 Token 管理

**存储结构**
```javascript
{
  key: 'authorization',
  value: 'Bearer xxx',
  expire: timestamp
}
```

**Token 有效期**
- 默认 3 天过期

### 6.2 鉴权流程

```
页面加载
    ↓
检查本地 authorization 是否存在
    ↓
调用 /user/info 验证 token 有效性
    ↓
success → 更新用户信息，显示已登录状态
    ↓
fail → 清除登录态，显示未登录状态
```

### 6.3 401 处理

当接口返回 401 状态码时：
1. 自动清除 authorization token
2. 自动清除 token
3. 自动清除 userInfo
4. 返回未登录状态

## 7. 样式规范

### 7.1 单位规范
- 使用 rpx 作为长度单位（适配不同屏幕）
- 禁止使用 px（固定像素）

### 7.2 命名规范
- CSS 类名使用 BEM 命名规范
- 组件命名使用 kebab-case

### 7.3 布局规范
- 使用 Flex 布局
- 禁止使用 float 布局
- 滚动区域使用 `scroll-y` 而非 scroll-view 组件
