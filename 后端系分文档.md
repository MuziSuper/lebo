# Lebo 小程序后端系统设计文档

## 1. 项目概述

### 1.1 项目背景
Lebo 是一个微信小程序后端服务系统，提供用户管理、商品管理、订单管理、积分体系等功能。采用微信云托管部署，前端通过 callContainer 直接调用后端 API，简化鉴权处理。

### 1.2 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| Spring Boot | 3.x | 核心框架 |
| JDK | 17 | 开发环境 |
| MyBatis-Plus | 3.5.x | ORM 框架 |
| MySQL | 8.x | 关系型数据库 |
| Lombok | - | 代码简化工具 |
| JWT | - | 身份认证 |
| 微信云托管 | - | 部署平台 |
| 微信云对象存储(COS) | - | 文件存储 |

## 2. 数据库设计

### 2.1 用户表 (user)

用户主表，存储用户基本信息。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| open_id | VARCHAR(255) | 主键，微信用户唯一标识 |
| session_key | VARCHAR(255) | 微信会话密钥（加密存储） |
| nick_name | VARCHAR(255) | 用户昵称（唯一） |
| union_id | VARCHAR(255) | 微信开放平台统一标识 |
| password | VARCHAR(255) | 密码（商户后台登录用） |
| avatar | VARCHAR(255) | 头像URL |
| gender | TINYINT | 性别：0-保密，1-男，2-女 |
| city | VARCHAR(255) | 城市 |
| age | INT | 年龄 |
| birthday | VARCHAR(255) | 生日（yyyy-MM-dd格式） |
| email | VARCHAR(255) | 邮箱 |
| phone | VARCHAR(255) | 手机号 |
| status | TINYINT | 状态：0-正常，1-不活跃，2-暂停，3-封禁，4-注销 |
| last_login | DATETIME(6) | 最后登录时间 |
| gmt_created | DATETIME(6) | 创建时间 |
| gmt_modified | DATETIME(6) | 修改时间 |
| is_deleted | TINYINT | 逻辑删除：0-正常，1-删除 |

```sql
CREATE TABLE `user` (
    `open_id` varchar(255) NOT NULL COMMENT '微信 openid',
    `session_key` varchar(255) DEFAULT NULL COMMENT '微信会话密钥',
    `nick_name` varchar(255) NOT NULL COMMENT '用户昵称',
    `union_id` varchar(255) DEFAULT NULL COMMENT '微信unionid',
    `password` varchar(255) DEFAULT NULL COMMENT '登录密码',
    `avatar` varchar(255) NOT NULL DEFAULT '' COMMENT '头像地址',
    `gender` tinyint NOT NULL DEFAULT '0' COMMENT '性别 0:保密 1:男 2:女',
    `city` varchar(255) DEFAULT NULL COMMENT '城市',
    `age` int DEFAULT NULL COMMENT '年龄',
    `birthday` varchar(255) DEFAULT NULL COMMENT '生日',
    `email` varchar(255) DEFAULT NULL COMMENT '邮箱',
    `phone` varchar(255) DEFAULT NULL COMMENT '手机号',
    `status` tinyint NOT NULL DEFAULT '0' COMMENT '用户状态 0:正常 1:不活跃 2:暂停 3:封禁 4:注销',
    `last_login` datetime(6) DEFAULT NULL COMMENT '最后登录时间',
    `gmt_created` datetime(6) DEFAULT CURRENT_TIMESTAMP(6) COMMENT '创建时间',
    `gmt_modified` datetime(6) DEFAULT CURRENT_TIMESTAMP(6) ON UPDATE CURRENT_TIMESTAMP(6) COMMENT '更新时间',
    `is_deleted` tinyint NOT NULL DEFAULT '0' COMMENT '逻辑删除 0:未删除 1:已删除',
    PRIMARY KEY (`open_id`),
    UNIQUE KEY `uk_nick_name` (`nick_name`),
    KEY `idx_status` (`status`),
    KEY `idx_phone` (`phone`),
    KEY `idx_gmt_created` (`gmt_created`),
    CONSTRAINT `user_chk_1` CHECK ((`status` between 0 and 4)),
    CONSTRAINT `user_chk_2` CHECK ((`gender` between 0 and 2))
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户信息表';
```

### 2.2 用户积分钱包表 (user_point)

存储用户积分信息。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | VARCHAR(255) | 主键，格式：UP_yyyyMMddHHmmss + 4位序列号 |
| open_id | VARCHAR(255) | 用户openid（唯一） |
| current_point | BIGINT | 当前可用积分 |
| accumulated_point | BIGINT | 累计获得积分 |
| is_deleted | INT | 逻辑删除：0-正常，1-删除 |
| gmt_created | DATETIME(6) | 创建时间 |
| gmt_modified | DATETIME(6) | 修改时间 |

```sql
CREATE TABLE `user_point` (
    `id` varchar(255) NOT NULL COMMENT '用户积分ID，格式：UP_yyyyMMddHHmmss + 4位序列号',
    `open_id` varchar(255) NOT NULL DEFAULT '' COMMENT '用户openid',
    `current_point` bigint NOT NULL DEFAULT '0' COMMENT '当前可用积分',
    `accumulated_point` bigint NOT NULL DEFAULT '0' COMMENT '累计获得积分',
    `is_deleted` int NOT NULL DEFAULT '0' COMMENT '逻辑删除 0:未删除 1:已删除',
    `gmt_created` datetime(6) DEFAULT CURRENT_TIMESTAMP(6) COMMENT '创建时间',
    `gmt_modified` datetime(6) DEFAULT CURRENT_TIMESTAMP(6) ON UPDATE CURRENT_TIMESTAMP(6) COMMENT '更新时间',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_open_id` (`open_id`),
    KEY `idx_current_point` (`current_point`),
    CONSTRAINT `user_point_chk_1` CHECK ((`current_point` >= 0))
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户积分账户表';
```

## 3. 核心枚举定义

### 3.1 订单状态枚举 (OrderTypeEnum)

| 代码 | 枚举值 | 说明 |
|------|--------|------|
| 1 | NONPAYMENT | 未支付 |
| 2 | PAID | 已支付 |
| 3 | FAILURE | 支付失败 |
| 4 | REFUNDED | 已退款 |
| 5 | OVER | 已结束 |

**订单状态流转说明**:

| 操作 | 原状态 | 目标状态 | 说明 |
|------|--------|----------|------|
| 创建订单 | - | NONPAYMENT | 用户创建订单，携带商品列表，填充订单金额 |
| 确认支付 | NONPAYMENT | PAID | 用户确认支付，填充支付方式、支付金额、支付时间 |
| 取消订单 | NONPAYMENT | FAILURE | 用户取消订单，填充结束时间、支付金额为0 |
| 订单超时 | NONPAYMENT | FAILURE | 订单创建超过5分钟未支付，自动取消 |
| 商家确认接单 | PAID | OVER | 商家确认接单，商品出库、积分入账、填充实际支付金额和结束时间 |
| 商家拒绝接单 | PAID | REFUNDED | 商家拒绝订单，库存退回，支付金额为0 |

**订单超时机制**:
- 使用 `ScheduledExecutorService` 线程池管理未支付订单
- 订单创建时加入超时队列，设置5分钟延迟任务
- 5分钟内用户支付或取消，从队列中移除
- 5分钟内无操作，自动执行超时逻辑，订单状态改为支付失败

## 4. ID生成策略

### 4.1 ID生成工具类 (IdUtil)

系统使用 `IdUtil` 工具类统一管理各类ID的生成，每种类型的ID拥有独立的序列号计数器，互不影响。

**ID格式**: `前缀_yyyyMMddHHmmss + 4位序列号`

| 实体类型 | 前缀 | 格式示例 | 生成方法 |
|---------|------|---------|---------|
| 订单 | `ORD_` | `ORD_202506021103020001` | `IdUtil.generateOrderId()` |
| 商品 | `PRO_` | `PRO_202506021103020001` | `IdUtil.generateProductId()` |
| 用户积分 | `UP_` | `UP_202506021103020001` | `IdUtil.generateUserPointId()` |
| 图片 | `IMAGE_` | `IMAGE_202506021103020001` | `IdUtil.generateImageId()` |
| 历史操作 | `HIS_` | `HIS_202506021103020001` | `IdUtil.generateHistoryOperationId()` |
| 出入库记录 | `INOUT_` | `INOUT_202506021103020001` | `IdUtil.generateInOutRecordId()` |

**特性**:
- 线程安全：使用 `synchronized` 关键字保证线程安全
- 独立计数：每种类型ID拥有独立的 `AtomicInteger` 序列号计数器
- 秒级重置：每秒最多生成 9999 个同类型ID
- 溢出处理：序列号超过 9999 时自动等待下一秒重新生成
