# Lebo 小程序后端系统设计文档

## 1. 项目概述

### 1.1 项目背景
Lebo 是一个微信小程序后端服务系统，提供用户管理、商品管理、订单管理、积分体系等功能。采用微信云托管部署，前端通过 callContainer 直接调用后端 API，简化鉴权处理。

### 1.2 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| Spring Boot | 3.x | 核心框架 |
| JDK | 17 | 开发环境 |
| MyBatis-Plus | 3.5.x | ORM 框架 |
| MySQL | 8.x | 关系型数据库 |
| Lombok | - | 代码简化工具 |
| JWT | - | 身份认证 |
| 微信云托管 | - | 部署平台 |
| 微信云对象存储(COS) | - | 文件存储 |

## 2. 数据库设计

### 2.1 用户表 (user)

用户主表，存储用户基本信息。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| open_id | VARCHAR(64) | 主键，微信用户唯一标识 |
| session_key | VARCHAR(128) | 微信会话密钥（加密存储） |
| nick_name | VARCHAR(100) | 用户昵称（唯一） |
| union_id | VARCHAR(128) | 微信开放平台统一标识 |
| password | VARCHAR(20) | 密码（商户后台登录用） |
| avatar | VARCHAR(500) | 头像URL |
| gender | INT | 性别：0-保密，1-男，2-女 |
| city | VARCHAR(100) | 城市 |
| age | INT | 年龄 |
| birthday | VARCHAR(20) | 生日（yyyy-MM-dd格式） |
| email | VARCHAR(100) | 邮箱 |
| phone | VARCHAR(20) | 手机号 |
| status | INT | 状态：0-活跃，1-不活跃，2-暂停，3-封禁，4-注销 |
| last_login | DATETIME | 最后登录时间 |
| gmt_created | DATETIME | 创建时间 |
| gmt_modified | DATETIME | 修改时间 |
| is_deleted | INT | 逻辑删除：0-正常，1-删除 |

```sql
CREATE TABLE `user` (
  `open_id` VARCHAR(64) NOT NULL COMMENT '微信用户唯一标识',
  `session_key` VARCHAR(128) COMMENT '微信会话密钥',
  `nick_name` VARCHAR(100) UNIQUE COMMENT '用户昵称',
  `union_id` VARCHAR(128) COMMENT '微信开放平台统一标识',
  `password` VARCHAR(20) COMMENT '密码',
  `avatar` VARCHAR(500) COMMENT '头像URL',
  `gender` INT DEFAULT 0 COMMENT '性别:0-保密,1-男,2-女',
  `city` VARCHAR(100) COMMENT '城市',
  `age` INT COMMENT '年龄',
  `birthday` VARCHAR(20) COMMENT '生日',
  `email` VARCHAR(100) COMMENT '邮箱',
  `phone` VARCHAR(20) COMMENT '手机号',
  `status` INT DEFAULT 0 COMMENT '状态:0-活跃,1-不活跃,2-暂停,3-封禁,4-注销',
  `last_login` DATETIME COMMENT '最后登录时间',
  `gmt_created` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `gmt_modified` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
  `is_deleted` INT DEFAULT 0 COMMENT '逻辑删除:0-正常,1-删除',
  PRIMARY KEY (`open_id`),
  UNIQUE KEY `uk_nick_name` (`nick_name`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

### 2.9 历史操作记录表 (history_operation)

记录商户操作历史，包括商品管理、分类管理、系统登录等操作。

| 字段名 | 类型 | 说明 |

**操作类型枚举**:

| 类型码 | 说明 |
|--------|------|
| 0 | 商品添加 |
| 1 | 商品修改 |
| 2 | 商品删除 |
| 3 | 手动备份数据 |
| 4 | 自动备份数据 |
| 5 | 后台系统登录 |
| 6 | 商品分类添加 |
| 7 | 商品分类修改 |
| 8 | 商品分类删除 |
| 9 | 导入备份数据 |

```sql
CREATE TABLE `history_operation` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `content` VARCHAR(500) COMMENT '操作内容',
```

```java
@Component
public class JwtAuthInterceptor implements HandlerInterceptor {
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                           HttpServletResponse response, 
                           Object handler) {
        // 1. 从请求头获取 Token
        String token = request.getHeader("Authorization");
        
        // 2. 验证 Token 有效性
        if (token == null || !jwtUtil.validateToken(token)) {
            response.setStatus(HttpStatus.UNAUTHORIZED.value());
            return false;
        }
        
        // 3. 解析 openid 并存入 ThreadLocal
        String openid = jwtUtil.getOpenIdFromToken(token);
        UserThreadUtil.setCurrentOpenId(openid);
        
        return true;
    }
}
```

```java
@Transactional(isolation = Isolation.REPEATABLE_READ)
public UserPoint updatePoint(String openid, Long point) {
    // 1. 加锁查询当前积分
    UserPoint userPoint = this.getOne(
        new QueryWrapper<UserPoint>()
            .eq("open_id", openid)
            .last("FOR UPDATE")
    );
    
    // 2. 检查积分是否足够（扣减时）
    long newPoint = userPoint.getCurrentPoint() + point;
    if (newPoint < 0) {
        throw new UserPointException("积分不足");
    }
    
    // 3. 更新积分（正数增加累计积分）
    if (point > 0) {
        userPoint.setAccumulatedPoint(
            userPoint.getAccumulatedPoint() + point
        );
    }
    userPoint.setCurrentPoint(newPoint);
    this.updateById(userPoint);
    
    return userPoint;
}
```

#### 4.4.4 订单结束流程（含积分结算）

```java
@Transactional(isolation = Isolation.REPEATABLE_READ)
public ResponseEntity<Result<Boolean>> orderOver(String orderId) {
    // 1. 查询订单并加锁
    Order order = this.getOne(
        new QueryWrapper<Order>()
            .eq("id", orderId)
            .last("FOR UPDATE")
    );
    
    // 2. 校验订单状态（只有已支付的订单才能结束）
    if (order.getPayType() != OrderTypeEnum.PAID) {
        throw new OrderException("订单状态异常");
    }
    
    // 3. 计算订单总积分
    List<OrderItem> items = orderItemService.listByOrderId(orderId);
    long totalPoint = 0L;
    for (OrderItem item : items) {
        Product product = productService.getById(item.getProductId());
        if (product != null && product.getPoint() != null) {
            totalPoint += product.getPoint() * item.getQuantity();
        }
    }
    
    // 4. 更新订单状态
    order.setPayType(OrderTypeEnum.OVER);
    order.setEndTime(LocalDateTime.now());
    this.updateById(order);
    
    // 5. 积分计入用户钱包
    if (totalPoint > 0) {
        userPointService.updatePoint(order.getOpenId(), totalPoint);
    }
    
    return response.setData(true);
}
```

```java
public List<Product> inOutBatch(List<ProductInOutDTO> dtoList) {
    List<Product> products = new ArrayList<>();
    
    for (ProductInOutDTO dto : dtoList) {
        // 1. 查询商品并加锁
        Product product = this.getOne(
            new QueryWrapper<Product>()
                .eq("id", dto.getProductId())
                .last("FOR UPDATE")
        );
        
        // 2. 更新库存
        long newStorage = product.getStorage() + dto.getNumber();
        if (newStorage < 0) {
            throw new ProductException("库存不足");
        }
        product.setStorage(newStorage);
        
        // 3. 更新商品状态
        if (newStorage == 0) {
            product.setStatus(ProductStatusEnum.SOLD_OUT);
        } else if (product.getStatus() == ProductStatusEnum.SOLD_OUT) {
            product.setStatus(ProductStatusEnum.SELL);
        }
        
        this.updateById(product);
        products.add(product);
        
        // 4. 创建出入库记录
        createInOutRecord(product, dto.getNumber());
    }
    
    return products;
}
```

```java
@Transactional(isolation = Isolation.REPEATABLE_READ, rollbackFor = Exception.class)
public void businessMethod() {
    // 使用 FOR UPDATE 加锁查询
    Entity entity = mapper.selectOne(
        new QueryWrapper<Entity>()
            .eq("id", id)
            .last("FOR UPDATE")
    );
    // 业务逻辑...
}
```

## 8. 安全设计

### 8.1 认证安全

- JWT Token 有效期：7天
- Token 存放在响应头 `Authorization` 中
- 敏感接口需要认证拦截器校验

### 8.2 数据安全

- `session_key` 使用 AES 加密存储
- 密码字段仅用于商户后台登录
- 支持逻辑删除，保护数据完整性

### 8.3 文件上传安全

- 目录名白名单校验（只允许字母、数字、下划线、连字符）
- 防止目录遍历攻击
- 文件类型根据扩展名自动设置
