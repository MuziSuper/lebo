# 微信云托管对象存储后端部署指南

## 一、概述

微信云托管对象存储是基于腾讯云COS（Cloud Object Storage）的文件存储服务，支持在服务端通过API进行文件的上传、下载和删除操作。

### 核心概念

1. **云环境ID (env)**: 每个微信云托管环境的唯一标识
2. **文件ID (file_id)**: 文件在云存储中的唯一标识，格式如 `cloud://xxx`
3. **access_token**: 接口调用凭证，用于鉴权
4. **cloudbase_access_token**: 微信云托管专属令牌，由微信后台自动推送

---

## 二、鉴权方式

### 方式一：开放接口服务（推荐）

**特点**: 完全免鉴权，无需维护任何token

**配置步骤**:
1. 登录微信云托管控制台
2. 进入「云调用」-「微信令牌权限配置」
3. 添加需要调用的接口路径：
   - 上传文件: `/tcb/uploadfile`
   - 下载文件: `/tcb/batchdownloadfile`
   - 删除文件: `/tcb/deletefile`
4. 打开「开放接口服务」开关
5. 重新构建服务版本使配置生效

**使用方式**:
- 请求时无需携带 `access_token` 或 `cloudbase_access_token`
- 使用 HTTP 协议（非 HTTPS）访问 `api.weixin.qq.com`
- 返回的 Header 中会包含 `x-openapi-seqid`，表示正在使用云调用链路

**请求示例**:
```http
POST http://api.weixin.qq.com/v2/tcb/uploadfile?env=your-env-id
Content-Type: application/json

{
  "path": "images/example.png"
}
```

### 方式二：微信令牌 (cloudbase_access_token)

**特点**: 微信后台自动推送，无需手动获取

**令牌信息**:
- 推送间隔: 10分钟
- 有效期: 30分钟
- 挂载路径: `/.tencentcloudbase/wx/cloudbase_access_token`

**使用方式**:
- GET参数名使用 `cloudbase_access_token` 而非 `access_token`
- 需要在控制台配置接口白名单

**请求示例**:
```http
POST https://api.weixin.qq.com/v2/tcb/uploadfile?cloudbase_access_token=YOUR_TOKEN&env=your-env-id
Content-Type: application/json

{
  "path": "images/example.png"
}
```

### 方式三：传统 access_token（不推荐）

**特点**: 需要使用 AppID 和 AppSecret 手动获取

**获取方式**:
```http
GET https://api.weixin.qq.com/cgi-bin/token?grant_type=client_credential&appid=APPID&secret=APPSECRET
```

## 三、上传文件

上传文件分为两个步骤：获取上传链接 → 上传文件到COS

### 步骤1：获取文件上传链接

**请求地址**:
```
POST https://api.weixin.qq.com/v2/tcb/uploadfile
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token/cloudbase_access_token | string | 是* | 接口调用凭证（开放接口服务模式不需要） |
| env | string | 是 | 云环境ID |
| path | string | 是 | 上传路径（不要以 / 开头） |

*注：使用开放接口服务时不需要传token

**返回参数**:

| 参数名 | 类型 | 说明 |
|--------|------|------|
| errcode | number | 错误码，0表示成功 |
| errmsg | string | 错误信息 |
| url | string | 上传URL（COS地址） |
| token | string | 临时token |
| authorization | string | 签名信息 |
| file_id | string | 文件ID，用于后续操作 |
| cos_file_id | string | COS文件ID |

**返回示例**:
```json
{
  "errcode": 0,
  "errmsg": "ok",
  "url": "https://cos.ap-shanghai.myqcloud.com/xxx",
  "token": "xxx",
  "authorization": "q-sign-algorithm=sha1&q-ak=xxx",
  "file_id": "cloud://xxx",
  "cos_file_id": "xxx"
}
```

### 步骤2：上传文件到COS

使用返回的信息构造 multipart/form-data 请求上传文件：

**请求地址**: 返回的 `url` 字段

**请求方式**: POST

**Content-Type**: multipart/form-data

**表单字段**:

| 字段名 | 值 | 说明 |
|--------|-----|------|
| key | 请求中的 path 字段 | 文件路径 |
| Signature | 返回的 authorization 字段 | 签名 |
| x-cos-security-token | 返回的 token 字段 | 临时token |
| x-cos-meta-fileid | 返回的 cos_file_id 字段 | **必填**，否则文件无法下载 |
| file | 文件二进制内容 | 文件内容 |

**重要提示**:
- `x-cos-meta-fileid` 字段必须填写，否则上传成功的文件下载时会报错
- 上传成功后无返回内容

**Java示例代码**:
```java
public String uploadFile(String env, String path, byte[] fileContent) throws Exception {
    // 步骤1: 获取上传链接
    String getUrl = "http://api.weixin.qq.com/v2/tcb/uploadfile?env=" + env;
    
    Map<String, String> params = new HashMap<>();
    params.put("env", env);
    params.put("path", path);
    
    String response = httpClient.post(getUrl, params);
    JSONObject json = JSON.parseObject(response);
    
    String uploadUrl = json.getString("url");
    String token = json.getString("token");
    String authorization = json.getString("authorization");
    String cosFileId = json.getString("cos_file_id");
    String fileId = json.getString("file_id");
    
    // 步骤2: 上传文件到COS
    MultipartEntityBuilder builder = MultipartEntityBuilder.create();
    builder.addTextBody("key", path);
    builder.addTextBody("Signature", authorization);
    builder.addTextBody("x-cos-security-token", token);
    builder.addTextBody("x-cos-meta-fileid", cosFileId);
    builder.addBinaryBody("file", fileContent, ContentType.DEFAULT_BINARY, path);
    
    HttpPost httpPost = new HttpPost(uploadUrl);
    httpPost.setEntity(builder.build());
    httpClient.execute(httpPost);
    
    return fileId;
}
```

## 四、获取文件下载链接

**请求地址**:
```
POST https://api.weixin.qq.com/v2/tcb/batchdownloadfile
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token/cloudbase_access_token | string | 是* | 接口调用凭证 |
| env | string | 是 | 云环境ID |
| file_list | Array | 是 | 文件列表（最多50个） |

**file_list 结构**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| fileid | string | 是 | 文件ID |
| max_age | number | 是 | 下载链接有效期（秒） |

**请求示例**:
```json
{
  "env": "your-env-id",
  "file_list": [
    {
      "fileid": "cloud://xxx",
      "max_age": 86400
    }
  ]
}
```

**返回示例**:
```json
{
  "errcode": 0,
  "errmsg": "ok",
  "file_list": [
    {
      "fileid": "cloud://xxx",
      "download_url": "https://xxx.cos.ap-shanghai.myqcloud.com/xxx",
      "status": 0,
      "errmsg": "ok"
    }
  ]
}
```

## 五、删除文件

**请求地址**:
```
POST https://api.weixin.qq.com/v2/tcb/deletefile
```

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| access_token/cloudbase_access_token | string | 是* | 接口调用凭证 |
| env | string | 是 | 云环境ID |
| fileid_list | Array | 是 | 文件ID列表（最多50个） |

**请求示例**:
```json
{
  "env": "your-env-id",
  "fileid_list": [
    "cloud://xxx",
    "cloud://yyy"
  ]
}
```

**返回示例**:
```json
{
  "errcode": 0,
  "errmsg": "ok",
  "delete_list": [
    "cloud://xxx",
    "cloud://yyy"
  ]
}
```

## 七、权限配置

### 控制台配置路径

1. 登录微信云托管控制台
2. 选择对应环境
3. 进入「云调用」-「微信令牌权限配置」
4. 添加接口路径（每行一个）：
   ```
   /tcb/uploadfile
   /tcb/batchdownloadfile
   /tcb/deletefile
   ```
5. 点击保存

### 开放接口服务开关

- 开启后，新创建的服务版本会自动具备云调用能力
- 已存在的服务版本需要重新构建才能生效
- 开关关闭期间创建的版本不具备云调用能力

## 八、资源复用场景

### 同主体资源复用

当环境共享给其他小程序/公众号使用时：

**使用 cloudbase_access_token**:
```
{授权方appid}@{cloudbase_access_token}
```

**示例**:
```
wx3937f3caaxxxxxx@CIIBEoAWVZM9yAA
```

### 使用开放接口服务

在请求地址后添加 `from_appid` 参数：
```
http://api.weixin.qq.com/v2/tcb/uploadfile?from_appid=wxbbbb&env=xxx
```

## 九、注意事项

### 1. 上传文件必须传 x-cos-meta-fileid

如果漏传此字段，文件看似上传成功但下载时会报错 `STORAGE_FILE_NONEXIST`，只能删除重传。

### 2. 跨域问题

- 本地调试时可能出现跨域问题
- 解决方案：在对象存储配置中添加安全域名

### 3. HTTPS vs HTTP

- 推荐使用 HTTP 协议以获得更好性能
- 如需使用 HTTPS，需要信任自签名证书：
  ```dockerfile
  # Dockerfile 中添加
  RUN apt-get update && apt-get install -y ca-certificates
  RUN update-ca-certificates
  ```

### 4. 请求限制

- 请求包体最大: 32 MiB
- 返回包体最大: 32 MiB
- 云调用最大超时: 30秒
- 下载文件一次最多: 50个
- 删除文件一次最多: 50个

### 5. 开放接口服务生效时机

- 开关开启后创建的版本才会具备云调用能力
- 已存在的版本需要重新构建

## 十、最佳实践

### 推荐架构

```
┌─────────────────┐
│   微信小程序     │
│  wx.cloud.init  │
└────────┬────────┘
         │ callContainer
         ▼
┌─────────────────┐
│  云托管服务容器  │
│  (开放接口服务)  │
└────────┬────────┘
         │ HTTP (免token)
         ▼
┌─────────────────┐
│ api.weixin.qq.com│
│   对象存储API    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   腾讯云COS      │
│   对象存储       │
└─────────────────┘
```

### 推荐配置

1. **开启开放接口服务**: 完全免鉴权，代码最简洁
2. **使用 HTTP 协议**: 性能更好
3. **保存 file_id**: 用于后续的下载和删除操作
4. **错误处理**: 检查 errcode 并进行相应处理

## 十一、与本地文件存储的对比

| 特性 | 本地文件存储 | 微信云托管对象存储 |
|------|-------------|-------------------|
| 持久性 | 容器重启丢失 | 永久保存 |
| 扩展性 | 受容器磁盘限制 | 无限制 |
| 访问方式 | 本地路径 | HTTP URL |
| 鉴权 | 无 | 需要token或云调用 |
| 适用场景 | 临时文件 | 用户上传文件、静态资源 |

---

## 十二、迁移建议

如果项目从本地文件存储迁移到微信云托管对象存储：

1. **配置开放接口服务**: 在控制台开启并配置权限
2. **修改上传逻辑**: 
   - 获取上传链接
   - 上传到COS
   - 保存返回的 file_id
3. **修改下载逻辑**: 
   - 使用 file_id 获取下载链接
   - 返回下载链接给前端
4. **修改删除逻辑**: 调用删除接口
5. **数据库字段调整**: 存储文件ID而非本地路径
